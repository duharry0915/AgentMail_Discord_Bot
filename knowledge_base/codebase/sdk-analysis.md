## 1. Overview & Architecture

Both SDKs are **Fern-generated** from a single API definition, meaning:

- All source files contain `This file was auto-generated by Fern from our API Definition`
- Direct PRs to generated code will not be merged — changes must go through the Fern spec
- Both SDKs share the same resource hierarchy, methods, and error semantics
- The Python SDK uses `snake_case`; the Node SDK uses `camelCase`

**Resource Hierarchy:**

```
Organization (top-level container)
└── Pod (optional multi-tenancy boundary)
    └── Inbox (email account, e.g., agent@agentmail.to)
        ├── Thread (conversation, auto-created)
        │   └── Message (individual email)
        │       └── Attachment
        └── Draft (unsent message)
```

**Client Architecture Pattern (both SDKs):**

```
Client (AgentMail / AgentMailClient)
├── inboxes           → InboxesClient
│   ├── .messages     → MessagesClient
│   ├── .threads      → ThreadsClient
│   ├── .drafts       → DraftsClient
│   └── .metrics      → MetricsClient
├── pods              → PodsClient
│   ├── .inboxes      → InboxesClient
│   ├── .threads      → ThreadsClient
│   ├── .drafts       → DraftsClient
│   ├── .domains      → DomainsClient
│   ├── .lists        → ListsClient
│   ├── .api_keys     → ApiKeysClient
│   └── .metrics      → MetricsClient
├── webhooks          → WebhooksClient
├── domains           → DomainsClient
├── api_keys          → ApiKeysClient
├── lists             → ListsClient
├── threads           → ThreadsClient (org-wide)
├── drafts            → DraftsClient (org-wide)
├── metrics           → MetricsClient
├── organizations     → OrganizationsClient
└── websockets        → WebsocketsClient
```

All sub-resource clients are **lazily initialized** as properties.

```python
from agentmail import AsyncAgentMail

client = AsyncAgentMail(
```


### 2.7 Error Handling

**Base error class:**

```python
from agentmail.core.api_error import ApiError

class ApiError(Exception):
    def __init__(self, *, headers=None, status_code=None, body=None):
        self.headers: Optional[Dict[str, str]]  # Response headers
        self.status_code: Optional[int]          # HTTP status code
        self.body: Any                           # ErrorResponse or raw body
```

**Specific error subclasses:**

| Error Class | Status Code | Import Path | Body Type |
|-------------|-------------|-------------|-----------|
| `ValidationError` | 400 | `agentmail.errors.ValidationError` | `ValidationErrorResponse` |
| `IsTakenError` | 403 | `agentmail.errors.IsTakenError` | `ErrorResponse` |
| `NotFoundError` | 404 | `agentmail.errors.NotFoundError` | `ErrorResponse` |
| `MessageRejectedError` | 403 | `agentmail.messages.MessageRejectedError` | `ErrorResponse` |

**Error handling example:**

```python
from agentmail import AgentMail
from agentmail.core.api_error import ApiError
from agentmail.errors import NotFoundError, ValidationError, IsTakenError

client = AgentMail()

try:
    inbox = client.inboxes.get(inbox_id="nonexistent@agentmail.to")
except NotFoundError as e:
    print(f"Not found: {e.body.message}")  # e.body is ErrorResponse
except ValidationError as e:
    print(f"Validation: {e.body.errors}")  # e.body is ValidationErrorResponse
except IsTakenError as e:
    print(f"Already taken: {e.body.message}")
except ApiError as e:
    print(f"API error {e.status_code}: {e.body}")
```

**ErrorResponse model:**

```python
class ErrorResponse(UncheckedBaseModel):
    name: ErrorName     # str, e.g. "NotFoundError"
    message: ErrorMessage  # str, human-readable description
```

**ValidationErrorResponse model:**

```python
class ValidationErrorResponse(UncheckedBaseModel):
    errors: Dict[str, List[str]]  # field → list of error messages
```

### 2.8 Retry & Timeout Configuration

**Default behavior:**

| Setting | Default | Override |
|---------|---------|---------|
| Timeout | 60 seconds | `timeout=N` on client, or `request_options={"timeout_in_seconds": N}` per request |
| Max retries | 2 | `request_options={"max_retries": N}` per request |
| Retried status codes | 408, 429, 5xx | Not configurable |
| Backoff strategy | Exponential with jitter | Not configurable |

**Per-request override:**

```python
client.inboxes.list(request_options={
    "timeout_in_seconds": 10,
    "max_retries": 0,  # Disable retries
})
```

**RequestOptions TypedDict:**

```python
class RequestOptions(TypedDict, total=False):
    timeout_in_seconds: int
    max_retries: int
    additional_headers: Dict[str, Any]
    additional_query_parameters: Dict[str, Any]
    additional_body_parameters: Dict[str, Any]
    chunk_size: int  # For streaming/download responses
```

### 2.9 WebSocket Support

**File:** `src/agentmail/websockets/client.py`

#### Sync WebSocket

```python
from agentmail import AgentMail, Subscribe, MessageReceivedEvent

client = AgentMail()

with client.websockets.connect() as socket:
    # Subscribe to specific inboxes
    socket.send_subscribe(Subscribe(
        inbox_ids=["agent@agentmail.to"]
    ))

    # Iterate over incoming events
    for event in socket:
        if isinstance(event, MessageReceivedEvent):
            print(f"New email: {event.message.subject}")
```

#### Async WebSocket

```python
from agentmail import AsyncAgentMail, Subscribe, MessageReceivedEvent

client = AsyncAgentMail()

async with client.websockets.connect() as socket:
    await socket.send_subscribe(Subscribe(
        inbox_ids=["agent@agentmail.to"]
    ))

    async for event in socket:
        if isinstance(event, MessageReceivedEvent):
            print(f"New email: {event.message.subject}")
```

#### Event Handler Pattern

```python
from agentmail.websockets import EventType

with client.websockets.connect() as socket:
    socket.on(EventType.OPEN, lambda _: print("Connected"))
    socket.on(EventType.MESSAGE, lambda msg: print(f"Event: {msg}"))
    socket.on(EventType.ERROR, lambda err: print(f"Error: {err}"))
    socket.on(EventType.CLOSE, lambda _: print("Disconnected"))

    # Start blocking listener (run in separate thread for sync)
    import threading
```

### 3.8 Retry & Timeout Configuration

**Default behavior:**

| Setting | Default | Override |
|---------|---------|---------|
| Timeout | 60 seconds | `timeoutInSeconds` on client or per-request |
| Max retries | 2 | `maxRetries` on client or per-request |
| Retried status codes | 408, 429, 5xx | Not configurable |
| Initial retry delay | 1000ms | Not configurable |
| Max retry delay | 60000ms | Not configurable |
| Jitter factor | ±20% | Not configurable |

**Retry delay calculation:**

1. Check `Retry-After` header (seconds or HTTP-date)
2. Check `X-RateLimit-Reset` header (Unix timestamp)
3. Exponential backoff: `1000ms * 2^attempt`
4. Apply ±20% symmetric jitter
5. Cap at 60 seconds

**AbortSignal support:**

## 7. Common Support Issues & Troubleshooting

### 7.1 Installation Issues

#### Python

**Problem: `pip install agentmail` fails with dependency conflicts**

```
ERROR: Cannot install agentmail because these package versions have conflicting dependencies.
```

**Solution:** Check for pydantic version conflicts. The SDK requires `pydantic>=1.9.2` and is compatible with both v1 and v2. If another package pins an incompatible pydantic version:

```bash
# Check current pydantic version
pip show pydantic

# Force upgrade/install in a virtual environment
python -m venv .venv
source .venv/bin/activate
pip install agentmail
```

**Problem: `ModuleNotFoundError: No module named 'agentmail'`**

Common causes:
1. Wrong Python environment — check `which python` / `which pip`
2. Installed globally but running in a venv (or vice versa)
3. Name conflict with another local `agentmail` directory

**Problem: WebSocket features don't work**

The `websockets` package (>=12.0) is required. If you installed agentmail without extras:

```bash
pip install agentmail  # websockets is a standard dependency, should be included
pip install websockets>=12.0  # Install explicitly if missing
```

#### Node.js

**Problem: `npm install agentmail` fails**

Requires Node.js >= 18.0.0. Check your version:
```bash
node --version
```

**Problem: ESM import issues**

The package supports both CJS and ESM. If you get import errors:

```javascript
// CommonJS
const { AgentMailClient } = require("agentmail");

// ESM
import { AgentMailClient } from "agentmail";
```

Make sure your `package.json` has `"type": "module"` for ESM, or use `.mjs` extension.

### 7.3 `client_id` Idempotency Issues

**Problem: Getting `IsTakenError` when creating inboxes/webhooks**

The `client_id` parameter ensures idempotency — if you create a resource with the same `client_id` twice, the second call returns the existing resource instead of creating a duplicate. However, if you try to create a resource with the same `client_id` but different parameters, you'll get an `IsTakenError`.

**Solution:**
```python
from agentmail.errors import IsTakenError

try:
    inbox = client.inboxes.create(
        request=CreateInboxRequest(
            username="my-agent",
            client_id="my-agent-inbox",
        )
    )
except IsTakenError:
    # The client_id already exists with different params
    # Either use a different client_id or fetch the existing resource
    pass
```

**Best practice:** Use deterministic `client_id` values based on your application's identity:
```python
client_id = f"user-{user_id}-primary-inbox"
```

### 7.4 Reply `to` Parameter Issues

**Problem: Reply goes to wrong address**

When using `reply()`, the SDK automatically determines the recipient from the original message. You do NOT need to specify `to`:

```python
# CORRECT - reply() handles 'to' automatically
client.inboxes.messages.reply(
    inbox_id="agent@agentmail.to",
    message_id="msg_xxx",
    text="My reply",
)

# WRONG - don't use send() for replies (breaks threading)
client.inboxes.messages.send(
    inbox_id="agent@agentmail.to",
    to=["user@example.com"],  # This creates a new thread!
    subject="Re: Original subject",
    text="My reply",
)
```

**Problem: Reply-all missing some recipients**

Use `reply_all()` / `replyAll()` instead of `reply()`:

```python
# Python
client.inboxes.messages.reply_all(
    inbox_id="agent@agentmail.to",
    message_id="msg_xxx",
    text="Reply to all participants",
)
```

```typescript
// Node.js
await client.inboxes.messages.replyAll("agent@agentmail.to", "msg_xxx", {
    text: "Reply to all participants",
});
```

### 7.5 Message Content Issues

**Problem: Email appears blank in recipient's inbox**

Always include both `text` AND `html` content:

```python
# CORRECT
client.inboxes.messages.send(
    inbox_id="agent@agentmail.to",
    to=["user@example.com"],
    subject="Hello",
    text="Plain text fallback",
    html="<p>Rich HTML content</p>",

# RISKY - some clients may show blank
client.inboxes.messages.send(
    inbox_id="agent@agentmail.to",
    to=["user@example.com"],
    subject="Hello",
    html="<p>HTML only</p>",  # No text fallback!
)
```

**Problem: Quoted reply text appears in `text`/`html` fields**

Use `extracted_text` / `extracted_html` for clean content without quoted reply chains:

```python
msg = client.inboxes.messages.get(inbox_id, message_id)
clean_reply = msg.extracted_text  # Just the new content, no quoted text
full_email = msg.text             # Full email including all quoted replies
```

### 7.6 Rate Limiting

**Problem: Getting 429 responses**

The SDK automatically retries 429 responses (up to `max_retries` times, default 2) with exponential backoff. If you're still hitting limits:

1. Reduce request frequency
2. Use webhooks/websockets instead of polling
3. Increase retry count for batch operations:

```python
client.inboxes.messages.send(
    ...,
    request_options={"max_retries": 5}
)
```

### 7.8 Domain Verification Issues

**Problem: Domain stuck in "pending" verification**

1. Download and verify DNS records:
   ```python
   domain = client.domains.get(domain_id="dom_xxx")
   for record in domain.verification_records:
       print(f"  Type: {record.type}")
       print(f"  Name: {record.name}")
       print(f"  Value: {record.value}")
       print(f"  Status: {record.status}")
       print()
   ```

2. **AWS Route 53 DKIM gotcha:** Long TXT records must be split into two quoted strings with NO space between them:
   ```
   CORRECT: "first-part""second-part"
   WRONG:   "first-part" "second-part"
   ```

3. **SPF record merging:** Only one SPF record per domain. Merge providers:
   ```
   v=spf1 include:spf.agentmail.to include:other-provider.com ~all
   ```

4. DNS propagation can take up to 48 hours.

### 7.9 Pod Lifecycle Issues

**Problem: Cannot delete pod**

Pods cannot be deleted while they contain resources:

```python
# Must delete all inboxes in the pod first
inboxes = client.pods.inboxes.list(pod_id="pod_xxx")
for inbox in inboxes.inboxes:
    client.pods.inboxes.delete(pod_id="pod_xxx", inbox_id=inbox.inbox_id)

# Then delete domains
domains = client.pods.domains.list(pod_id="pod_xxx")
for domain in domains.domains:
    client.pods.domains.delete(pod_id="pod_xxx", domain_id=domain.domain_id)

# Now delete the pod
client.pods.delete(pod_id="pod_xxx")
```

**Problem: Cannot move inbox between pods**

Inboxes cannot be moved. Create a new inbox in the target pod instead.

### 7.11 Bounce/Complaint Issues

**Problem: `MessageRejectedError` when sending**

AgentMail permanently blocks sending to addresses that have bounced or filed complaints. This protects your sender reputation.

```python
from agentmail.messages import MessageRejectedError

try:
    client.inboxes.messages.send(
        inbox_id="agent@agentmail.to",
        to=["bounced@example.com"],
        subject="Hello",
        text="Hi!",
    )
except MessageRejectedError as e:
    print(f"Rejected: {e.body.message}")
    # Check your blocklist
    blocked = client.lists.list(direction="send", type="block")
```

**Prevention:** Keep bounce rate below 4% to avoid account review.
